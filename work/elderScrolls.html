The Elder
Scrolls VI

I asked the ES community which game had their favorite UI and why. The answers split pretty evenly between Oblivion and Morrowind, but the complaints were consistent across the board. Skyrim was console-ported and needed SkyUI to actually be usable. Oblivion had charm but buried everything in submenus. Morrowind put everything in one view but was chaos if you didn't already know what you were doing.

With ES6 still years away, I took that as a brief and designed the whole thing myself. Every screen, every flow, every interaction. The goal was simple: keep the identity and soul that makes Elder Scrolls feel like Elder Scrolls, but actually make it work.

Research

Before designing anything, I asked the Elder Scrolls community a direct question: which game had your favorite UI and why? The feedback was split between Oblivion and Morrowind, but the pain points were consistent. Here are four that shaped the design decisions most.

"Skyrim's default UI is so poorly optimised for PC. I have never gone back since I got SkyUI." ThirdStreetSeren, r/ElderScrolls
Design response

The tab bar navigation and keyboard-first layout means the ES6 UI works properly on PC without needing a mod to fix it. No console-port compromises.

"Oblivion is a submenu fest that takes a while to get back used to." ThirdStreetSeren, r/ElderScrolls
Design response

Every screen is one tab away from any other. There are no nested menus. You open the pause menu and everything is already visible across the top bar.

"I liked being able to see every magic effect on me at a glance in Morrowind, without going several pages into the menu like Skyrim." lydiardbell, r/ElderScrolls
Design response

Active effects are shown directly on the Character screen with timers visible. No extra clicks, no digging through menus mid-combat to see what is debuffing you.

"Oblivion has so much character. Skyrim's UI has the aesthetics but no soul. Bethesda should hire whoever did SkyUI." bmgarcia20 + InBlurFather, r/ElderScrolls
Design response

The UI uses a consistent visual language rooted in ES lore, from the diamond selection icon to the aged parchment map. It should feel like it belongs in this world, not borrowed from another game.

Pain Points

The community was consistent. Across Oblivion, Skyrim, and Morrowind the same frustrations kept surfacing — console-ported controls, buried menus, lost identity. These six pain points from player feedback shaped every design decision.

P-01 Critical
Console-First Design
Skyrim's UI was built for controller and never properly adapted for PC. Players needed a mod just to make the inventory usable on keyboard and mouse.
P-02 Critical
Submenu Overload
Oblivion buried everything in nested menus. Players had to navigate three or four layers deep to reach basic information they needed regularly.
P-03 Critical
Lost Identity
Skyrim went so minimal it lost the character Oblivion had. The community consistently described it as having aesthetics but no soul — generic rather than ES-specific.
P-04 High
Hidden Active Effects
Morrowind showed every active effect at a glance. Skyrim hid this behind multiple menu levels, so players couldn't quickly see what was debuffing them mid-combat.
P-05 Medium
No Consistent System
Each ES game had a completely different UI language with no visual continuity. Players switching between titles had to relearn navigation from scratch every time.
P-06 Medium
Readability Under Pressure
Oblivion's ornate design was beautiful but hard to parse quickly. Players needed to distinguish actionable elements from decoration in seconds — the UI made that difficult.

User Flow

The full prototype covers two main flows: the main menu flow from launch to gameplay, and the in-game pause menu system. Every screen connects to every other screen it should logically connect to. The diagram below maps the complete navigation structure.

Main Menu

The main menu is quiet on purpose. Elder Scrolls menus usually work best when they feel like a door into the world, not a product screen. I used a darker illustrated backdrop, a low-contrast red-brown wash, and one simple vertical menu so the first interaction feels calm and old-world.

The press-start state keeps the title centered, then moves into the functional menu once the player enters. It gives the game a proper title beat without making the menu itself harder to use.

01 — Launch Flow

Press Start

What it is. A title beat with the logo, one start prompt, and a dark illustrated backdrop.

Why it looks this way. The logo gets room before the practical menu appears. The prompt is small and centered so the screen feels ceremonial, not like a settings page.

Issue it solves. It gives the game a stronger first impression without adding clutter or extra choices before the player has even started.

02 — Launch Flow

Main Menu

What it is. A left-aligned menu for Continue, Load, Settings, Credits, and Exit.

Why it looks this way. The serif type, selection diamond, and warm red-brown wash keep it old-world, while the spacing makes the list easier to scan than older dense menus.

Issue it solves. The menu keeps the Elder Scrolls mood without making the first screen feel heavy or overbuilt.

03 — Launch Flow

Loading Screen

What it is. A wide landscape loading state with a thin progress bar and one gameplay tip.

Why it looks this way. The art does most of the work. The UI sits low and quiet so it feels like travel through the world instead of a system screen.

Issue it solves. It gives feedback and a useful tip without covering the scenery with noisy interface elements.

In-Game UI

The HUD needed to stay out of the way. Skyrim was strong at restraint, but weak at surfacing what mattered. I kept the world mostly untouched, then used thin compass lines, bottom resource bars, and a compact quest tracker that only takes the top-right corner.

The visual style is intentionally worn and quiet. It has metal, parchment, and old fantasy detailing, but the system information still reads cleanly during movement.

01 — Gameplay HUD

Exploration HUD

What it is. The active gameplay HUD with compass, quest tracker, objective steps, health, magicka, stamina, and a small active-effect marker.

Why it looks this way. Permanent UI stays on the edges. The center is kept clean for the world, while the bars use familiar Elder Scrolls color language.

Issue it solves. The player gets the essentials without the HUD fighting the environment or forcing constant pauses to check status.

02 — Gameplay HUD

Quest + Combat Readability

What it is. The same HUD tested in a darker exploration scene with weapon, shield, compass, quest list, and resource bars visible.

Why it looks this way. The interface stays thin and anchored to the frame so it survives busy landscapes without turning into a full overlay.

Issue it solves. It keeps quest direction and player status readable in darker scenes, where subtle fantasy UI can easily disappear.

Pause Menus

The pause system is where most of the project lives. The problem with older Elder Scrolls menus was never one single screen. It was the way everything stacked: inventory in one place, character in another, effects hidden elsewhere, quests split into nested pages, and settings feeling like they belonged to a different game.

I used one top navigation bar for the full pause layer: Inventory, Character, Map, Quests, Skills, Settings. Each tab changes the content, but the frame stays consistent. It feels more modern, but still keeps the weight and texture of Elder Scrolls.

01 — Pause Tab

Inventory

What it is. A category inventory with item rows in the center and a full item detail panel on the right.

Why it looks this way. The left icons give quick category jumps, while the center table keeps name, damage, weight, and value lined up for scanning.

Issue it solves. It avoids the slow single-list problem and gives players a proper scan-and-compare layout.

02 — Pause Tab

Character

What it is. A character overview for race, level, XP, core attributes, build identity, skill totals, resistances, and active effects.

Why it looks this way. Main stats sit in the center, and the right-side cards hold secondary information. Active effects are visible without opening another page.

Issue it solves. Players can understand their build and current status in one read instead of checking separate stat and magic-effect screens.

03 — Pause Tab

Map

What it is. A parchment-style Hammerfell map with the current objective pinned to the left.

Why it looks this way. The map is treated like an object from the world, not a satellite view. The top navigation stays system-level, but the map itself feels hand-drawn.

Issue it solves. Location and quest context stay together, so the player knows where they are going and why.

04 — Pause Tab

Quests

What it is. A quest journal with filters, selected quest text, objectives, rewards, giver, region, status, and a journal entry.

Why it looks this way. Browsing, reading, and reference info each get their own area: list on the left, story in the center, details on the right.

Issue it solves. Quest tracking stops being a thin checklist. The task, reward, and story are all readable without digging.

05 — Pause Tab

Skill Tree

What it is. A full skill tree view with Combat, Exploration, Stealth, Gathering, and Crafting shown side by side.

Why it looks this way. I used columns instead of deep categories so the player can compare paths at a glance. The selected skill still gets a detail card in the corner.

Issue it solves. Leveling feels less buried. Players can see the shape of their build and spend points without entering a chain of submenus.

06 — Pause Layer

System Overlay

What it is. The pause system over a blurred version of the live game, with Continue, Load/Save, Settings, Photo Mode, Quit, and Exit.

Why it looks this way. The world stays present, but softened. The left menu handles system actions while the top bar keeps the main tabs consistent.

Issue it solves. Pausing does not feel like being pulled into a dead black menu. The player always knows where they came from.

Settings & Save

Settings and save screens usually get the least care, but players spend real time there. I kept them inside the same pause language so they feel like part of the game instead of a separate utility menu.

The important choice was context. The game world stays blurred behind the panel, the left pause menu stays visible, and the player always knows how to get back.

01 — System Screen

Settings

What it is. A gameplay settings panel with difficulty, aim assist, subtitles, brightness, motion blur, audio, and sensitivity controls.

Why it looks this way. Controls are long rows with values on the right, so keyboard and mouse users can move quickly without tiny nested widgets.

Issue it solves. Settings stay readable and consistent with the pause system instead of feeling like a separate launcher menu.

02 — System Screen

Load / Save

What it is. A save browser with screenshot thumbnails, location, character, level, playtime, and date.

Why it looks this way. The thumbnail leads each row because memory is visual. Players usually remember where a save happened before they remember the timestamp.

Issue it solves. It removes the guesswork from picking a save slot and makes load/save feel like part of the adventure record.
