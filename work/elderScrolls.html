The Elder
Scrolls VI

I asked the ES community which game had their favorite UI and why. The answers split pretty evenly between Oblivion and Morrowind, but the complaints were consistent across the board. Skyrim was console-ported and needed SkyUI to actually be usable. Oblivion had charm but buried everything in submenus. Morrowind put everything in one view but was chaos if you didn't already know what you were doing.

With ES6 still years away, I took that as a brief and designed the whole thing myself. Every screen, every flow, every interaction. The goal was simple: keep the identity and soul that makes Elder Scrolls feel like Elder Scrolls, but actually make it work.

Research

Before designing anything, I asked the Elder Scrolls community a direct question: which game had your favorite UI and why? The feedback was split between Oblivion and Morrowind, but the pain points were consistent. Here are four that shaped the design decisions most.

"Skyrim's default UI is so poorly optimised for PC. I have never gone back since I got SkyUI." ThirdStreetSeren, r/ElderScrolls
Design response

The tab bar navigation and keyboard-first layout means the ES6 UI works properly on PC without needing a mod to fix it. No console-port compromises.

"Oblivion is a submenu fest that takes a while to get back used to." ThirdStreetSeren, r/ElderScrolls
Design response

Every screen is one tab away from any other. There are no nested menus. You open the pause menu and everything is already visible across the top bar.

"I liked being able to see every magic effect on me at a glance in Morrowind, without going several pages into the menu like Skyrim." lydiardbell, r/ElderScrolls
Design response

Active effects are shown directly on the Character screen with timers visible. No extra clicks, no digging through menus mid-combat to see what is debuffing you.

"Oblivion has so much character. Skyrim's UI has the aesthetics but no soul. Bethesda should hire whoever did SkyUI." bmgarcia20 + InBlurFather, r/ElderScrolls
Design response

The UI uses a consistent visual language rooted in ES lore, from the diamond selection icon to the aged parchment map. It should feel like it belongs in this world, not borrowed from another game.

User Flow

The full prototype covers two main flows: the main menu flow from launch to gameplay, and the in-game pause menu system. Every screen connects to every other screen it should logically connect to. The diagram below maps the complete navigation structure.

Main Menu

A lot of people in the community pointed to Oblivion's character as something Skyrim lost. The paper texture, the hand-drawn feel, the sense that the interface actually belonged in that world. Skyrim went minimalist and clean, which works fine, but it ended up feeling like a generic RPG menu rather than an Elder Scrolls one.

For the main menu I wanted the world to do the talking. The press start screen drops you straight into the atmosphere with just the title and a single prompt. From there the main menu keeps navigation tucked to the left so the environment fills everything else. The diamond icon you see on the selected item shows up on every screen throughout the UI as a consistent selection marker.

In-Game UI

Skyrim gets credit for a clean HUD and it deserves it. But one thing that kept coming up in the community was how Morrowind actually handled active effects better. In Skyrim, if something is debuffing you mid-combat you have to go three levels into a menu to find out what. Morrowind showed everything at a glance.

The ES6 HUD keeps Skyrim's restraint. Bars at the bottom, compass and current objective top-right. But active effects are visible without opening anything. There are two HUD states here: one with a Khajiit character in bright daylight, and one first-person view in a snowy mountain setting. Both show the same system in different contexts.

Pause Menus

Even people who loved Oblivion's look admitted the menus were a submenu fest. And Morrowind's floating windows were flexible but overwhelming unless you already knew the game. The ES6 pause system tries to take the best from both without the downsides of either.

One tab bar across the top covers everything: Inventory, Character, Map, Quests, Skills, Settings. One button to open, one button to switch. No nested menus to dig through. Every screen shares the same layout structure so moving between them feels natural. Each interaction has a sound effect tied to it, which is something the vanilla ES games have always underused.

A few things worth calling out individually. The inventory has categories down the left side with item stats and a detail panel on the right, so you never have to hunt for information. The character screen has Active Effects visible directly on the page, no clicking required. The quest log separates Main, Side, Faction, and Completed quests with filters at the top, and each quest shows a journal entry alongside the objectives. And yes, one of the quests is called "Hire Me" with quest giver Fady Neddar. The load and save screen shows a screenshot thumbnail per save slot, something the series has never had natively and should have had years ago.

Settings & Save

Settings and load/save are the screens most games treat as afterthoughts, and you can tell. Here they follow the exact same structure as everything else. Same layout, same icon system, same transitions. The settings panel opens as an overlay over the pause menu rather than replacing it entirely, which keeps the context intact. Small thing, but it matters.
